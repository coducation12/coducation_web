# Step 3: 강사 및 커리큘럼 관리 기능 고도화 계획

## 1. 개요
기존에 유실되었던 강사 관리와 수업 시간표 기능을 복구하되, 단순 복구가 아닌 관리 효율성을 높이는 방향으로 개선(고도화)합니다. 관리자가 학원 운영을 한눈에 파악하고 제어할 수 있도록 직관적인 UI/UX를 제공하는 것이 목표입니다.

## 2. 상세 개발 내용

### A. 실시간 알림 시스템 (Telegram Bot Webhook)
*   **목표**: 상담문의 발생 시 관리자에게 즉각적인 알림 전송하여 대응 속도 향상.
*   **주요 기능**:
    1.  **텔레그램 봇 연동**: 상담 신청 완료(`addConsultation`) 시 실시간 메시지 발송.
    2.  **알림 내용**: 신청자 성명, 연락처, 희망 과정, 문의 내용을 포함한 요약 메시지.
    3.  **관리 링크**: 메시지 내에 관리자 페이지 상담 관리 탭으로 즉시 이동 가능한 링크 포함.

### B. 강사 관리 시스템 (Teacher Management)
*   **목표**: 강사 정보의 체계적 관리 및 학생 배정 효율화
*   **주요 기능**:
    1.  **강사 목록 카드 뷰 (Card View)**
        *   강사 사진, 이름, 주 담당 과목, 연락처를 카드 형태로 직관적으로 표시.
        *   활성/비활성 상태 표시 (재직/휴직 등).
    2.  **강사 상세 프로필 (Detail & Edit)**
        *   **기본 정보**: 이름, 연락처, 이메일, 생년월일.
        *   **전문 이력**: 약력(Career), 자격증(Certificates) 목록 관리 (태그 형태 또는 리스트 형태).
        *   **소개(Bio)**: 학원 홈페이지 강사 소개란에 노출될 멘트 관리.
        *   **프로필 이미지**: `ImageUpload` 컴포넌트를 활용한 사진 관리.
    3.  **학생 배정 관리 (Assignment)**
        *   해당 강사가 담당하는 학생 목록 조회.
        *   드래그 앤 드롭 또는 멀티 셀렉트로 학생을 강사에게 손쉽게 배정/해제 기능.

### C. 커리큘럼 및 수업 시간표 (Curriculum & Timetable)
*   **목표**: 수업 일정의 시각화 및 커리큘럼 체계 확립
*   **주요 기능**:
    1.  **주간 시간표 (Weekly Timetable)**
        *   **캘린더 UI**: 요일별/시간대별 수업 현황을 캘린더 형태로 시각화.
        *   **수업 블록**: 각 수업 블록에 과목명, 담당 강사, 강의실 표시.
        *   **인터랙션**: 시간표 블록 클릭 시 상세 정보 팝업 (수강 학생 목록 등).
    2.  **커리큘럼 상세 관리 (Curriculum Detail)**
        *   **레벨별 관리**: 기초/중급/고급 과정별 커리큘럼 그룹화 (이미 요청주신 컨텐츠 관리의 교육과정 탭과 데이터 연동).
        *   **세부 진도표**: 각 커리큘럼별 세부 챕터(Chapter) 및 진도율 체크리스트 관리.

### D. 데이터베이스 구조 (예상)
*   `teachers` 테이블: 사용자(`users`) 테이블과 1:1 매핑 또는 별도 확장 테이블.
*   `classes` (또는 `timetables`) 테이블: 수업 시간, 요일, 강사ID, 커리큘럼ID 연결.
*   `student_teachers`: 학생-강사 매핑 테이블 (N:M 관계).

## 3. 작업 순서
1.  **실시간 알림 시스템 구현**: 텔레그램 봇 API 연동 및 상담문의 시 알림 발송 로직 추가.
2.  **DB 스키마 점검**: 강사 및 시간표 관련 테이블 존재 여부 확인 및 마이그레이션.
3.  **강사 관리 페이지 개발**: 목록 -> 등록 -> 상세/수정 순으로 구현.
4.  **시간표 페이지 개발**: 주간 캘린더 UI 구현 및 데이터 연동.
5.  **메인 연동**: 강사 소개 데이터가 메인 페이지에도 반영되도록 연결.
